Documentazione tecnica

Guida completa ad ARchetipo.

ARchetipo è un sistema di sei skill che governa il ciclo di vita iniziale di un progetto software: dalla discovery del prodotto fino all'implementazione, con agenti virtuali specializzati a ogni fase. Pensato per sviluppatori che vogliono trasformare un AI coding agent in un team di prodotto reale — concreto, senza fronzoli.

In breve. ARchetipo è un set di skill markdown che trasforma un AI coding agent (Claude Code, Codex, Gemini CLI, OpenCode, Copilot) in un team di prodotto. Le skill leggono .archetipo/config.yaml per sapere dove salvare gli artefatti, poi operano astrattamente sui connector.

1 · Il progetto

ARchetipo è un framework di skill per AI coding agent che governa il ciclo di vita iniziale di un progetto software. Sei skill concatenate in pipeline coprono l'intero percorso: dalla discovery del prodotto e la definizione del PRD, fino alla creazione del backlog, alla pianificazione tecnica e all'implementazione del codice. Ogni skill coinvolge un team virtuale di specialisti che collaborano sotto la direzione dell'agente. Gli artefatti prodotti vengono salvati su file system locale in Markdown o su GitHub Projects v2.

2 · Installazione

Un unico script installa tutto: scarica le skill, mostra un menu interattivo per selezionare gli AI tool di destinazione, copia ogni skill nella directory corretta e crea .archetipo/config.yaml.

terminalbash
# macOS / Linux
curl -fsSL https://raw.githubusercontent.com/techreloaded-ar/ARchetipo/main/install.sh | bash
PowerShellps1
# Windows
irm https://raw.githubusercontent.com/techreloaded-ar/ARchetipo/main/install.ps1 | iex

Cosa fa l'installer

  • Scarica le skill ufficiali dal repository GitHub.
  • Mostra un menu interattivo per scegliere su quali AI tool installarle.
  • Copia ciascuna skill nella directory corretta del tool selezionato.
  • Crea .archetipo/config.yaml con il connector scelto (file di default).

3 · Prerequisiti

  • macOS / Linux: curl e unzip (inclusi di default nella maggior parte delle distribuzioni).
  • Windows: PowerShell 5.1 o superiore.
  • Opzionale: gh CLI autenticato con permessi repo + project, solo se pianifichi di usare il connector GitHub.

4 · Quickstart end-to-end

Dopo l'installazione, segui questo flusso nel tuo AI coding agent. Ogni slash-command invoca una skill che produce artefatti persistenti e alimenta la fase successiva.

AI agentcommands
# 1 · Definisci il prodotto
> /archetipo-inception
  → docs/PRD.md

# 2 · (opzionale) Concept visivi
> /archetipo-design
  → docs/mockups/

# 3 · Genera il backlog iniziale
> /archetipo-spec
  → docs/BACKLOG.md

# 4 · Pianifica una storia
> /archetipo-plan US-001
  → docs/planning/US-001.md

# 5 · Implementa
> /archetipo-implement US-001
  → codice + test + code review
  → storia transita in REVIEW

# Alternativa · esecuzione autonoma di più storie
> /archetipo-autopilot
  → avanzamento coordinato del backlog
Auto-detect della lingua. Ogni skill adotta automaticamente la lingua della conversazione. Scrivi in italiano, ricevi artefatti in italiano. Scrivi in inglese, ricevi artefatti in inglese.

5 · Glossario

Concetti di base che ricorrono in tutta la documentazione e nelle skill.

TermineSignificato
PRDProduct Requirements Document — definisce cosa costruire, per chi e perché. È il punto di partenza della pipeline.
BacklogLista ordinata di user story da implementare, ciascuna con priorità, stima e stato (TODO → PLANNED → IN PROGRESS → REVIEW → DONE).
User storyUnità atomica di requisito dal punto di vista dell'utente: "Come <persona> voglio <azione> così che <beneficio>".
EpicRaggruppamento di user story correlate con un obiettivo condiviso. Ogni story appartiene a un epic.
Planning fileDocumento tecnico per una singola story con soluzione, task ordinati, dipendenze e strategia di test. Salvato in docs/planning/US-XXX.md.
INVESTCriteri di qualità per le user story: Indipendente, Negoziabile, Valorosa, Estimabile, Small, Testabile.
Acceptance criteriaCondizioni verificabili che definiscono quando una story può considerarsi completata.
BackendModalità di storage degli artefatti del progetto: file system locale (Markdown) oppure GitHub Projects v2 + Issues.

6 · La pipeline

Le sei skill si concatenano in una sequenza orientata alla delivery. Ogni skill produce artefatti consumati dalla successiva. ARchetipo implementa la metodologia Spec-Driven Development: la specifica è il contratto, e il ciclo spec → plan → implement si ripete per ogni slice di valore fino a completare il prodotto.

Stati workflow

Le storie transitano tra stati standardizzati. Le transizioni automatiche avvengono durante il loop; il passaggio finale a DONE è sempre manuale.

StatoSignificatoTransizione
TODOStoria nel backlog, non ancora lavorataauto (da spec)
PLANNEDPianificazione tecnica completataauto (da plan)
IN PROGRESSImplementazione in corsoauto (da implement)
REVIEWCode review + test passati, pronta per approvazione umanaauto (fine implement)
DONEStoria accettata e rilasciatamanuale

Le sei skill

01 · archetipo-inception

Discovery e definizione del prodotto. Guida la fase di discovery con un team virtuale. Raccoglie obiettivi, scope e ipotesi di business, produce il PRD.

  • Output: docs/PRD.md
  • Prerequisito: nessuno
  • Team: Andrea, Costanza, Leonardo, Livia, Emanuele

02 · archetipo-design OPZIONALE

Mockup e prototipi UI. Produce mockup HTML/CSS isolati in docs/mockups/ senza toccare il codice applicativo. Usati come riferimento visivo da archetipo-implement.

  • Output: docs/mockups/
  • Prerequisito: nessuno
  • Team: Livia

03 · archetipo-spec

Creazione ed estensione del backlog. Converte il PRD in backlog oppure aggiunge nuove story INVEST-compliant a uno esistente, con domande sfidanti di qualità. Auto-rileva la modalità.

  • Output: docs/BACKLOG.md
  • Prerequisito: PRD esistente
  • Team: Andrea, Emanuele

04 · archetipo-plan

Piano tecnico di implementazione. Trasforma una story TODO in piano tecnico con soluzione, task ordinati, dipendenze e strategia di test (inclusi e2e con video recording).

  • Output: docs/planning/US-XXX.md
  • Prerequisito: story in BACKLOG.md
  • Team: Emanuele, Leonardo, Ugo, Mina

05 · archetipo-implement

Scrittura di codice e test. Esegue il piano tecnico: scrive codice, test unit/integration/e2e, effettua code review automatica e porta la story verso REVIEW.

  • Output: codice + test + backlog aggiornato
  • Prerequisito: planning file (story PLANNED)
  • Team: Ugo, Mina, Cesare

⚡ archetipo-autopilot MODALITÀ AUTOMATICA

Esecuzione autonoma del ciclo plan → implement. Coordina automaticamente plan e implement su più story in sequenza usando subagent isolati. Configurabile per epic, priorità, numero massimo di story e strategia sugli errori. Alternativa al ciclo manuale.

  • Output: avanzamento coordinato del backlog
  • Prerequisito: story TODO/PLANNED nel backlog
  • Team: orchestratore (nessun ruolo dedicato)
Skill sperimentali. La cartella skills-extra/ contiene skill riservate a versioni future (integrazioni con Figma Make, Vibe Kanban, e altre in lavorazione). Non fanno parte del flusso ufficiale.

7 · Da dove parto?

Guida rapida per scegliere la skill giusta in base alla fase del tuo progetto. Segui le domande in ordine.

  • Hai già un PRD? Se no → parti da archetipo-inception per condurre la discovery e produrre il documento.
  • Vuoi concept visivi prima dello sviluppo? Se sì → usa archetipo-design per mockup HTML/CSS isolati in docs/mockups/.
  • Hai un backlog con user story? Se no, hai solo il PRD → usa archetipo-spec per generare backlog strutturato con epic e story INVEST-compliant.
  • Le story sono già pianificate (PLANNED)? Se no, sono in TODO → usa archetipo-plan per produrre il piano tecnico con task, dipendenze e strategia di test.
  • Hai più story pronte da processare in autonomia? Se sì → archetipo-autopilot esegue il ciclo plan→implement su più story con subagent isolati e state file per riprendere se interrotto. Se no, una alla volta → archetipo-implement.

8 · Backend supportati

ARchetipo supporta due modalità di storage, configurabili in .archetipo/config.yaml tramite il campo backend: file | github. Le skill non gestiscono mai direttamente la persistenza: il flusso è sempre lo stesso.

  • La skill legge .archetipo/config.yaml per sapere quale connector è attivo.
  • La skill carica .archetipo/contracts.md (catalogo delle operazioni).
  • La skill carica .archetipo/connectors/{file|github}.md (implementazione).
  • La skill chiama operazioni astratte come READ: fetch_backlog_items, WRITE: save_plan, WRITE: transition_status.

Risultato: puoi cambiare dove vivono i tuoi artefatti senza cambiare una riga nelle skill.

Default · file

File system locale

  • Artefatti in Markdown locale
  • Backlog in docs/BACKLOG.md
  • Piani in docs/planning/US-XXX.md
  • Mockup in docs/mockups/US-XXX/
  • Nessuna autenticazione, funziona offline
  • Ideale per progetti solo o early-stage
Cloud · github

GitHub Projects v2

  • Story come GitHub Issues con labels
  • Piano nel corpo dell'issue
  • Sub-issues per ogni task eseguibile
  • Transizioni di stato come campi custom del Project
  • Richiede gh auth refresh -s project
  • Ideale per team distribuiti con stakeholder visibili sul tracking
Contratto completo. Il catalogo delle operazioni supportate da ogni connector è in .archetipo/contracts.md. Se vuoi scrivere un connector custom (Linear, Jira, Trello) parti da lì.

9 · Configurazione

Dopo l'installazione, .archetipo/config.yaml contiene tutti i parametri del progetto. È versionato nel repo e può essere modificato a mano.

.archetipo/config.yamlyaml
connector: file              # file | github

paths:
  prd:          docs/PRD.md
  backlog:      docs/BACKLOG.md   # solo per connector file
  planning:     docs/planning/
  mockups:      docs/mockups/
  test_results: docs/test-results/

workflow:
  statuses:
    todo:        TODO
    planned:     PLANNED
    in_progress: IN PROGRESS
    review:      REVIEW
    done:        DONE               # transizione manuale

github:                      # solo per connector github
  # owner:          auto-detected
  # project_number: auto-detected

10 · Lingua e conversazione

Le skill rilevano automaticamente la lingua del messaggio utente e adattano tutti gli artefatti di conseguenza. Questo include:

  • Testo del PRD, delle user story e dei piani tecnici.
  • Commit message e descrizioni di PR.
  • Copy dei mockup generati da archetipo-design.

I nomi delle persone virtuali (Andrea, Costanza, Emanuele, Leonardo, Ugo, Mina, Cesare, Livia) restano invariati — sono identità, non traduzioni.

11 · Filosofia

  • Il team è una lente, non un costume. Ogni persona applica un angolo di analisi. Il brief è conciso e serve a rendere visibile il processo.
  • Contesto lean. Le skill leggono solo ciò che serve, nell'ordine: shared-runtime → contracts → connector → template. I file grandi si leggono chirurgicamente.
  • Output persistenti. Ogni fase produce artefatti che vivono nel repo o nel sistema connector. Il prossimo comando parte da lì.
  • Autonomia responsabile. Le skill si fermano solo davanti a blocker reali (dipendenze esterne, ambiguità sul contratto). Adattamenti locali e fix meccanici non richiedono conferma.
  • Tool e connector agnostici. Cambiare AI agent o sistema di tracking non deve riscrivere il processo.

12 · FAQ

Serve sempre il PRD per iniziare?

Per costruire un backlog solido sì: il PRD è il punto di partenza ideale. Se non esiste ancora, usa archetipo-inception per crearlo prima di passare a archetipo-spec.

Posso aggiungere story senza rifare tutto il backlog?

Sì. archetipo-spec distingue tra generazione iniziale ed estensione: se il backlog esiste, aggiunge o raffina story senza ripartire da zero.

Quando ha senso usare autopilot invece del ciclo manuale?

Quando hai più story già ordinate nel backlog e vuoi delegare il ciclo operativo agli agenti. Il percorso manuale è migliore quando vuoi revisionare ogni piano prima dell'implementazione.

I mockup prodotti da design finiscono nel codice applicativo?

No. archetipo-design produce file isolati in docs/mockups/ usati solo come riferimento visivo da archetipo-implement, che li legge prima di scrivere l'interfaccia reale.

Posso usare ARchetipo su un progetto già esistente?

Sì. archetipo-spec in modalità extend-backlog aggiunge storie a un backlog esistente. archetipo-plan e archetipo-implement lavorano su qualsiasi storia presente, indipendentemente da come è stata creata.

Cosa succede se il mio AI tool non supporta i subagent?

Le skill principali funzionano lo stesso, in modalità in-context. Solo archetipo-autopilot richiede subagent isolati, perché orchestra più passi per storia.

13 · Troubleshooting

La skill non trova il PRD

Verifica che paths.prd in .archetipo/config.yaml punti al file corretto. Il valore di default è docs/PRD.md.

Il connector GitHub non transiziona le issue

Controlla che gh sia autenticato (gh auth status) e che il token abbia gli scope repo e project. Le transizioni di stato avvengono tramite GraphQL sui campi custom del Project.

Gli artefatti sono nella lingua sbagliata

Le skill rilevano la lingua dall'ultimo messaggio utente. Se il contesto è misto, invia un messaggio esplicito nella lingua desiderata prima di invocare la skill.

Debug di una skill

Ogni skill dichiara quali reference carica. Attiva la modalità verbose del tuo AI tool e controlla che le operazioni connector (READ:, WRITE:) siano eseguite nell'ordine atteso. .archetipo/contracts.md è la fonte di verità.
