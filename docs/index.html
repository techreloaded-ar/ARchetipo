Perché esiste

Gli AI agent sono potenti. Manca il processo.

Prompt isolati non costruiscono prodotti. ARchetipo introduce un flusso ispirato ai team reali, con ruoli specializzati e artefatti persistenti — PRD, backlog, piani, mockup — che passano da una fase all'altra senza perdersi.

01 · METODO

Un processo, non un prompt.

Dalla discovery al code review, ogni fase ha la propria skill, i propri ruoli e i propri output. Il ciclo spec → plan → implement si ripete per ogni slice di valore finché il prodotto non è pronto.

02 · PORTABILITÀ

Agnostico rispetto al tool.

Le stesse skill funzionano su Claude Code, Codex, Gemini CLI, OpenCode e GitHub Copilot. Cambi agent, non cambi metodo.

03 · PERSISTENZA

File markdown o GitHub Projects.

Gli artefatti vivono dove vuoi — nel repo o come issue — senza cambiare il modo in cui parli alle skill.

04 · AUTONOMIA

Guidato quando serve, autopilot quando conviene.

Le skill si fermano solo davanti a blocker reali. Fix meccanici, adattamenti locali e aggiornamenti di test non richiedono conferma. L'intero backlog può girare in autopilot.

Le skill

Ogni skill è utilizzabile in isolamento, ma insieme formano il ciclo completo di consegna di una storia. Rilevano la lingua della conversazione e producono artefatti persistenti.

Architettura connector

Le skill non toccano mai la persistenza.

Un contratto astratto (READ:, WRITE:) separa la logica delle skill dal sistema che conserva gli artefatti. Oggi file e github. Domani Linear, Jira, o il tuo connector custom.

.archetipo/config.yamlyaml
connector: file              # file | github

paths:
  prd:      docs/PRD.md
  backlog:  docs/BACKLOG.md
  planning: docs/planning/
  mockups:  docs/mockups/
  test_results: docs/test-results/

workflow:
  statuses:
    todo:        TODO
    planned:     PLANNED
    in_progress: IN PROGRESS
    review:      REVIEW
    done:        DONE
Default · file

Versionato con il tuo repo

  • Backlog in un singolo file markdown
  • Piani tecnici in docs/planning/US-XXX.md
  • Zero dipendenze esterne

FAQ

Le domande che ci fanno sempre.

Se manca qualcosa, apri una discussion su GitHub — rispondiamo in giornata.

Devo usare tutte le skill? +

No. Ogni skill è utilizzabile in isolamento. Puoi partire direttamente da archetipo-plan se hai già un backlog scritto a mano, o usare solo archetipo-design per esplorare concept visivi.

Posso usare ARchetipo su un progetto già esistente? +

Sì. archetipo-spec in modalità extend-backlog aggiunge storie a un backlog esistente. archetipo-plan e archetipo-implement lavorano su qualsiasi storia presente, indipendentemente da come è stata creata.

Cosa succede se il mio AI tool non supporta i subagent? +

Le skill principali funzionano lo stesso, in modalità in-context. Solo archetipo-autopilot richiede subagent isolati, perché orchestra più passi per storia.

Gli artefatti sono vincolati a un formato? +

I template in references/ sono opinati ma modificabili. Se usi il connector github, le issue seguono un layout preciso — vedi .archetipo/connectors/github.md.

Perché i nomi italiani? +

ARchetipo nasce in Italia e mantiene una voce riconoscibile: un team con nomi propri funziona meglio di "the analyst agent" / "the architect agent". Le skill parlano comunque nella lingua della conversazione (auto-detected).

Come si fa il debug di una skill? +

Ogni skill dichiara quali reference carica. Attiva la modalità verbose del tuo AI tool e controlla che le operazioni connector (READ:, WRITE:) siano eseguite nell'ordine atteso. .archetipo/contracts.md è la fonte di verità.
